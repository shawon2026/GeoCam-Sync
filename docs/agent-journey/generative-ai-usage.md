# Generative AI Journey

## Overview

This project was built with iterative AI-assisted planning and execution.  
The final direction came from repeated prompt refinement and human decisions at each step.

## Gemini Contributions

- Helped shape the project naming direction and initial framing.
- Supported early architecture journey thinking (how to phase the implementation).
- Assisted with launcher icon creation workflow and asset direction.
- Helped with UI idea exploration (how the app should feel and look at a high level).

## ChatGPT Contributions

- Helped draft the project architecture map.
- Helped evaluate package choices (what to use, where to use, and why).
- Helped with implementation approach guidance for feature modules and flow design.

## IDE Agent (Codex) Contributions

- Generated and refactored boilerplate code across modules.
- Assisted in repetitive engineering tasks (cleanup, consistent structure updates, repeated edits).
- Implemented UI scaffolding and repeated UI wiring tasks.
- Applied project-wide structural changes quickly while keeping code compile-safe.

## Human Direction and Control

- Folder architecture rules were explicitly provided by the user.
- Coding style, professionalism level, and quality expectations were user-driven.
- The user continuously refined prompts to improve output quality.
- AI suggestions were accepted or rejected based on user intent and project requirements.

## Prompt Iteration Pattern

The journey was not one-shot.  
It required repeated prompt updates to:

- clarify architecture boundaries,
- improve package decisions,
- tighten coding standards,
- and align outputs with professional delivery expectations.

## Final Note

AI was used as an accelerator for planning and execution.  
Core technical direction, constraints, and final decision-making stayed with the user.
