# Language Policy

## Strict Rule

All repository content must be written in English:

- README files
- Documentation under `docs/`
- Code comments and inline notes
- Skill/workflow guidance files
- Script output/help text
- Commit messages and PR descriptions

## Exception

Localization resource values (for example Bengali text inside `intl_bn.arb`) are allowed because they are product translations, not engineering documentation.
Locale-specific data constants (for example Bengali numerals used in conversion utilities) are also allowed when required by product behavior.
